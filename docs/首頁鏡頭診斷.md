# 首頁看不到鏡頭 - 快速診斷指南

## 🔍 可能原因與解決方案

### 1️⃣ **最常見：瀏覽器快取問題**
**解決方法：**
```
按 Ctrl + Shift + R（強制刷新）
```

### 2️⃣ **JavaScript 錯誤**
**檢查方法：**
1. 按 **F12** 開啟開發者工具
2. 切換到 **Console** 標籤
3. 重新整理頁面
4. 查看是否有**紅色錯誤訊息**

**常見錯誤：**
- `togglePauseCamera is not defined` - 代碼有問題
- `Failed to fetch` - API 無法連接
- `Uncaught SyntaxError` - JavaScript 語法錯誤

### 3️⃣ **API 端點問題**
**檢查方法：**
1. 按 **F12** → **Network** 標籤
2. 重新整理頁面
3. 找到 `/api/cameras` 請求
4. 查看狀態碼：
   - ✅ **200** = 正常
   - ❌ **404** = 路由不存在
   - ❌ **500** = 伺服器錯誤

**解決方法：**
如果看到錯誤，請檢查伺服器控制台輸出

### 4️⃣ **攝影機未初始化**
**檢查伺服器控制台：**
應該看到：
```
攝影機 Camera1 初始化成功
攝影機 Camera2 初始化成功
```

如果看到錯誤：
- `無法開啟攝影機` - 硬體問題
- `設定檔不存在` - config.ini 問題

### 5️⃣ **Web 應用未正確啟動**
**確認啟動成功：**
控制台應該顯示：
```
* Running on http://127.0.0.1:5000
* Debug mode: on
```

## 🚀 快速修復步驟

### Step 1: 強制刷新
```
Ctrl + Shift + R
```

### Step 2: 檢查開發者工具
```
F12 → Console → 看是否有紅色錯誤
```

### Step 3: 檢查 Network
```
F12 → Network → 重新整理 → 找 /api/cameras
```

### Step 4: 重啟伺服器（如果需要）
```
關閉 start_web_app.bat
重新執行 start_web_app.bat
等待 "Running on" 訊息出現
```

## 📝 如果還是看不到

請提供以下資訊：
1. **瀏覽器 Console 的錯誤訊息**（F12 → Console）
2. **Network 標籤中 /api/cameras 的狀態**（F12 → Network）
3. **伺服器控制台的輸出**（start_web_app.bat 視窗）

這樣我可以精確診斷問題！
